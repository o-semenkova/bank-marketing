# 🏦 Bank Marketing: Передбачення підписки на строковий депозит

## Опис задачі

Задача бінарної класифікації: передбачити, чи оформить клієнт банку строковий депозит за результатами телефонної маркетингової кампанії.

Дані стосуються прямих маркетингових кампаній (телефонних дзвінків) португальської банківської установи. Датасет містить **41 188 записів** та **20 ознак** — демографічні дані клієнтів, параметри контакту, історію попередніх кампаній та макроекономічні показники.

**Датасет:** [Bank Additional Full (Kaggle)](https://www.kaggle.com/datasets/sahistapatel96/bankadditionalfullcsv)

---

## Що було зроблено

### 1. Exploratory Data Analysis

**Числові змінні:**
- Виявлено сильну мультиколінеарність між 5 макроекономічними змінними (VIF до 25 000). Ітеративне VIF-видалення визначило оптимальний набір для лінійних моделей: `emp.var.rate` та `cons.conf.idx`
- Виявлено J-подібний нелінійний зв'язок віку з конверсією: 65+ → 46.8%, 17-25 → 20.9%, 36-45 → 8.5%
- `pdays` має спеціальне кодування (999 = не було контакту), потребує трансформації
- `duration` виключена з моделі через data leakage

**Категоріальні змінні (Cramér's V):**
- `poutcome` — найсильніший предиктор (V ≈ 0.33): success → 65% конверсії
- `month` та `contact` — помірний зв'язок (V ≈ 0.13–0.14)
- `housing`, `loan`, `day_of_week` — слабкий зв'язок з цільовою змінною

**Взаємодія змінних:**
- "Успішні" місяці (mar, sep, oct, dec) збігаються з періодами економічного спаду — `month` і макрозмінні несуть частково однакову інформацію

### 2. Вибір метрик

| Метрика | Роль | Обґрунтування |
|---------|------|---------------|
| **F1 (yes)** | Основна | Баланс precision/recall для мінорітарного класу (11.3%) |
| **ROC-AUC** | Додаткова | Порівняння моделей незалежно від порогу класифікації |
| Precision | Допоміжна | Частка результативних дзвінків серед рекомендованих |
| Recall | Допоміжна | Частка знайдених потенційних клієнтів |

Accuracy не підходить через дисбаланс класів: модель, що завжди каже "no", має accuracy 88.7%.

### 3. Препроцесинг

Виконано окремий препроцесинг для лінійних та нелінійних моделей:

| Крок | Лінійні (LogReg, kNN) | Нелінійні (Tree, XGBoost) |
|------|----------------------|--------------------------|
| `duration` | Видалено | Видалено |
| `pdays` | was_contacted_before + заміна 999→-1 | was_contacted_before + заміна 999→-1 |
| Групування | basic.4y/6y/9y → basic; default yes+unknown | basic.4y/6y/9y → basic; default yes+unknown |
| Макрозмінні | Лише 2 (VIF) | Усі 5 |
| Нові ознаки | age² | — |
| Кодування | One-Hot (drop_first) | OrdinalEncoder |
| Масштабування | StandardScaler | Не потрібне |
| Outliers | campaign cap 95-й перцентиль | Без обробки |

### 4. Моделі та результати

| # | Модель | F1 (test) | AUC (test) | Precision | Recall | Коментар |

### 5. Тюнінг гіперпараметрів XGBoost

Два підходи з 50 ітераціями кожен:

- **RandomizedSearchCV** — випадковий пошук. Знайшов параметри з хорошим train-результатом, але з перенавчанням (F1 train=0.61 vs test=0.50)
- **Hyperopt (Bayesian Optimization, TPE)** — розумний пошук, кожна ітерація враховує попередні. Знайшов більш збалансовані параметри: F1 train≈test, найвищий AUC=0.82

**Висновок:** Bayesian Optimization ефективніший за ту саму кількість ітерацій — знаходить параметри, які краще узагальнюються.

### 6. Feature Importance та SHAP

**Feature Importance (XGBoost):** `nr.employed` домінує (0.66) через те, що XGBoost обирає її як основну змінну для сплітів серед корельованих макрозмінних.

**SHAP-аналіз** дає більш збалансовану картину:

| Ознака | Mean \|SHAP\| | Інтерпретація |
|--------|-------------|---------------|
| nr.employed | 0.44 | Менше зайнятих → більше підписок (період спаду) |
| emp.var.rate | 0.18 | Негативний темп зайнятості → більше підписок |
| contact | 0.12 | Cellular значно ефективніший за telephone |
| euribor3m | 0.10 | Низькі ставки → більше підписок |
| cons.conf.idx | 0.09 | Індекс довіри споживачів |
| age | 0.07 | Молодь і пенсіонери конвертуються краще |

SHAP розкрив, що `contact` і `age` значно важливіші, ніж показувала Feature Importance.

### 7. Аналіз помилок

**FN (340 пропущених клієнтів):** мають профіль "типових відмовників" (висока зайнятість, немає попереднього контакту), але дуже тривалу розмову (медіана 785 сек). Модель не має доступу до `duration`, тому не може вловити ці випадки.

**FP (773 хибних рекомендацій):** мають "ідеальний" макроекономічний профіль (спад, низькі ставки), але короткий дзвінок (медіана 155 сек) — клієнт не зацікавився.
